Установка и настройка аутентификации

В этом разделе вы найдете пошаговые инструкции и примеры кода для реализации JWT и OAuth 2.0 в ваших проектах. Мы рассмотрим настройку серверной и клиентской частей для популярных языков программирования и фреймворков.

Настройка JWT

Для реализации JWT мы будем использовать популярные библиотеки для разных языков программирования. Ниже приведены примеры для Node.js и Python.

1. Node.js (jsonwebtoken)

Установите библиотеку jsonwebtoken и настройте сервер для создания и проверки JWT. 

npm install jsonwebtoken

Пример серверного кода:

const express = require('express');
const jwt = require('jsonwebtoken');

const app = express();
app.use(express.json());

const JWT_SECRET = process.env.JWT_SECRET;

app.post('/login', (req, res) => {
  const { username, password } = req.body;

  if (username === 'user' && password === 'pass') {
    const token = jwt.sign(
      { username, role: 'user' },
      JWT_SECRET,
      { expiresIn: '1h' }
    );
    return res.json({ token });
  }

  return res.status(401).json({ message: 'Неверные учетные данные' });
});
Рекомендуется хранить JWT в HTTP-only cookies.

2. Python (PyJWT)

pip install PyJWT
from flask import Flask, request, jsonify
import jwt

app = Flask(__name__)
JWT_SECRET = 'your-secret-key'

@app.route('/login', methods=['POST'])
def login():
    data = request.get_json()

    if data['username'] == 'user' and data['password'] == 'pass':
        token = jwt.encode(
            {'username': data['username'], 'role': 'user'},
            JWT_SECRET,
            algorithm='HS256'
        )
        return jsonify({'token': token})

    return jsonify({'message': 'Неверные учетные данные'}), 401

Настройка OAuth 2.0

1. Регистрация приложения в Google

  • Создайте учетные данные OAuth 2.0 Client ID
  • Укажите redirect URI http://localhost:3000/auth/callback
  • Получите client_id и client_secret
Никогда не публикуйте client_secret в клиентском коде.

2. Node.js (passport-google-oauth20)

npm install passport

Используйте переменные окружения для хранения секретов.

Дополнительные рекомендации

  • Безопасное хранение секретов: Используйте переменные окружения.
  • HTTPS: Всегда используйте HTTPS.
  • Обработка ошибок: Обрабатывайте случаи истечения токенов.
  • Логирование: Отслеживайте попытки аутентификации.
  • Тестирование: Тестируйте интеграцию локально.
